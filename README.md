## 1. Core Philosophy & The Typhon Metaphor

Humanitas Primoris ("Humanity First") rejects all socioeconomic and technological conditions that seek to commodify, track, and hollow out human existence. We actively combat "bullshit jobs," algorithmic authoritarianism, and the techno-gnostic fantasy of escaping the flesh (digital uploads, simulated presence, and billionaire escapism). We embrace *Amor Mundi* (Radical Love of the World), human frailty, and mortality as the absolute conditions of meaning.

### The Typhon Motif
We have adopted the mythological figure of **Typhon** as the ultimate allegory for the systemic forces arrayed against us. Typhon represents the chaos of fascism, nationalism, algorithmic manipulation, and the toxic bigotries polluting modern systems. 

Drawing from Plato’s *Phaedrus*, we contrast this monstrous, furious engine with the "gentler and simpler creature" of authentic human solidarity. This movement serves as the **Seal of Binding**. Just as Typhon was imprisoned beneath Mount Etna, our collective, decentralized infrastructure is the cultural and structural weight engineered to choke out and contain this subterranean hatred.

## 5. Operational Security (OPSEC) Protocol

All contributors and the Scribe operate under non-negotiable security guardrails to maintain complete anonymity and network isolation:

1. **The Clean Room:** All repository management, code deployment, and updates must occur strictly within an isolated Ubuntu Virtual Machine using native Tor services routed through the Tor Browser.
2. **Web GUI Manipulation:** Edits to files must be committed directly via the hosting provider web GUIs inside the Tor browser to prevent local machine footprint profiling or SSH key correlation.
3. **Anonymized Account Creation:** All service access tokens, emails (ProtonMail), and hosting accounts must be initialized over Tor. Out-of-band verifications must utilize public internet connections entirely disconnected from the operator's home IP (e.g., public library networks).
4. **Outbound Notification Ban:** The use of Discord webhooks, Slack APIs, or automated third-party build status pings is strictly prohibited to mitigate correlation attacks.
5. **Inbound Tracking:** Repository telemetry is consumed passively via standard RSS/Atom feeds, pulled locally inside the secure VM using localized clients (e.g., GNOME Feeds).
